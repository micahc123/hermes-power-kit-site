The problem

Four things that make people quit Hermes.

Each is widely reported in public Hermes issue threads. The kit is built to remove all four.

💸 It quietly burns tokens

A large share of every call is fixed overhead before you type a word, and long sessions silently replay their whole history — with no native cost visibility.

→ Tuned config + a live /cost view + spend caps

🧠 The memory forgets you

The headline "memory that grows with you" is also its most-reported failure — files silently overflow limits, and a corrupted store can wipe recall.

→ Structured memory + integrity check & auto-backup

⚙️ Setup fights newcomers

Hard to install for non-engineers, and a known routing quirk can bill you on a provider you never chose.

→ One-command install that pins your provider

🛡️ The skill hubs are risky

Public skill hubs are full of unvetted — and reportedly outright malicious — skills. Installing blind from them is a real supply-chain risk.

→ A hand-vetted catalog + a preview-only installer

How it works

It bolts onto Hermes — no fork, no patches.

Everything drops into the files and primitives Hermes already uses. You can add or remove any piece at any time, and uninstall the whole thing by deleting a few files.

📂

Native surfaces

A tuned ~/.hermes/config.yaml, operating rules in AGENTS.md, skills in the open SKILL.md standard, branded slash-commands as bundles, scheduled work via Hermes cron/heartbeat, and parallel work via its delegate_task subagents.

🧱

Layered & opt-in

A Core everyone installs (cost, memory, safety) plus packs you add à la carte with kit add <pack>. Non-coders run one command and get everything sensible; power users pick.

⌨️

One CLI runs it

kit status · kit doctor (self-test) · kit add/remove <pack> · kit loop to create and manage budget-capped automation.

✅

Built to hold up

Pure bash + standard-library Python — zero third-party dependencies to break on a user's machine. Backed by an automated test suite and safety-first defaults that confirm before anything destructive.

Get started

Up and running in one command.

Clone the kit, run the installer, and you're done — it sets up Hermes if needed, pins your provider, and drops the Core into ~/.hermes. Then add the packs you want.

$ git clone https://github.com/micahc123/hermes-power-kit.git && cd hermes-power-kit $ bash install.sh $ kit add research loops routines catalog $ kit doctor # confirm everything's healthy
